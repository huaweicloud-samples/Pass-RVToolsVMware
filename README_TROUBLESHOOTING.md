# RVTools分析工具 - 故障排除指南

## 🔧 快速修复说明

### 问题1：选择文件后没有任何变化

**原因分析：**
文件选择事件可能没有被正确触发。

**解决方案：**

1. **打开诊断工具**
   - 在浏览器中打开 `diagnostic.html` 文件
   - 这将帮助我们检测问题所在

2. **测试步骤**
   ```
   步骤1: 查看环境检测结果
   - 应该显示所有库(✅)都已加载
   
   步骤2: 点击"测试文件选择"按钮
   - 应该弹出文件选择对话框
   - 选择Excel文件后应该显示文件信息
   
   步骤3: 如果测试2成功，点击"测试Excel解析"
   - 应该显示工作表信息
   ```

3. **查看浏览器控制台**
   - 按 F12 打开开发者工具
   - 切换到 Console（控制台）标签
   - 查看是否有JavaScript错误信息
   - 截图发给开发者

### 问题2：文件验证失败

**症状：** 弹出"❌ 文件验证失败"提示

**解决方案：**
- 确保文件扩展名是 `.xlsx` 或 `.xls`
- 确保文件不是损坏的
- 尝试用Excel重新保存文件

### 问题3：上传后一直显示"正在解析"

**可能原因：**
- 文件太大
- 网络问题导致CDN库加载失败
- Excel文件格式不兼容

**解决方案：**
1. 检查网络连接
2. 查看控制台是否有错误
3. 尝试用浏览器打开 index.html（不是从文件资源管理器双击）

## 📝 正确打开文件的方式

### 方法1：使用浏览器打开
```
不要直接双击 index.html 文件

正确做法：
1. 右键点击 index.html
2. 选择 "打开方式"
3. 选择 "Google Chrome" 或 "Microsoft Edge"
```

### 方法2：使用VS Code Live Server
```
如果你使用VS Code：
1. 安装 Live Server 扩展
2. 右键点击 index.html
3. 选择 "Open with Live Server"
```

### 方法3：从命令行打开
```
在文件所在目录打开命令行：
- Windows: 在地址栏输入 cmd 回车
- 输入: start index.html
```

## 🔍 诊断工具使用说明

打开 `diagnostic.html` 会显示：

### 1. 环境检测
检查所有必需的JavaScript库是否正确加载：
- ✅ FileReader API
- ✅ XLSX库（Excel解析）
- ✅ Chart.js（图表）
- ✅ jsPDF（PDF生成）

### 2. 文件上传测试
手动测试文件选择功能

### 3. Excel解析测试
读取Excel文件并显示工作表信息

### 4. 主应用测试
打开主分析工具

## 📞 获取帮助

如果以上方法都不能解决问题：

1. **截图诊断页面** - 发送给开发者
2. **打开控制台** - 按F12，截图Console标签页内容
3. **尝试不同的浏览器** - Chrome、Edge、Firefox

## ✅ 成功标志

当你成功上传文件后，应该看到：

1. **弹出确认框** - 显示文件名和大小
2. **上传区域消失** - 显示加载动画
3. **分析结果显示** - 出现统计卡片和分析图表

如果没有看到这些，说明有问题需要排查。
